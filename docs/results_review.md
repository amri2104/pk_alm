# Results Review

This note reviews the outputs generated by:

```bash
python examples/full_alm_reporting_workflow.py
```

The example workflow uses the required live AAL/ACTUS service-backed asset
path. It writes Full ALM reporting artifacts to `outputs/full_alm_scenario/`.
It does not modify the protected Stage-1 baseline outputs in
`outputs/stage1_baseline/`.

## Generated Output Inventory

CSV outputs:

- `full_alm_kpi_summary.csv`
- `full_alm_annual_cashflows.csv`
- `full_alm_cashflow_by_source.csv`
- `full_alm_net_cashflow.csv`
- `full_alm_combined_cashflows.csv`
- `full_alm_funding_ratio_trajectory.csv`

PNG outputs:

- `plots/funding_ratio_trajectory.png`
- `plots/net_cashflow.png`
- `plots/cashflow_by_source.png`
- `plots/assets_vs_liabilities.png`

No `benchmark_plausibility.csv` was generated by the default example run
because no caller-provided benchmark reference values were passed.

## CSV Review

- `full_alm_kpi_summary.csv`: one-row summary of headline ALM indicators.
  In the generated run, the initial funding ratio is 107.60%, the final and
  minimum funding ratio are about 89.53%, and five projection years are below
  100%.
- `full_alm_annual_cashflows.csv`: annual liquidity table for reporting
  years 2026 to 2037. It includes contribution cashflows, pension payments,
  capital withdrawals, other cashflows, net cashflow, structural net cashflow,
  cumulative net cashflow, and currency.
- `full_alm_cashflow_by_source.csv`: annual payoff totals split by source.
  It separates `BVG` liability-side cashflows from `ACTUS` asset-side
  cashflows.
- `full_alm_net_cashflow.csv`: compact plot-ready table focused on annual
  net cashflow and structural net cashflow.
- `full_alm_combined_cashflows.csv`: canonical `CashflowRecord` table with
  the combined BVG and ACTUS/AAL asset cashflow events.
- `full_alm_funding_ratio_trajectory.csv`: funding-ratio path by projection
  year, including assets, the deterministic Stage-1 liability proxy, funding
  ratio, funding-ratio percent, and currency.

## PNG Review

- `funding_ratio_trajectory.png`: best thesis figure for the funding path.
  It shows the decline from the initial funding ratio to the final/minimum
  level in the deterministic example run.
- `net_cashflow.png`: best thesis figure for liquidity pressure. It compares
  annual net cashflow with structural net cashflow.
- `cashflow_by_source.png`: useful supporting figure for explaining the
  bridge between BVG liability cashflows and ACTUS/AAL asset cashflows.
- `assets_vs_liabilities.png`: useful supporting figure for explaining the
  numerator and denominator behind the funding-ratio trajectory.

## Recommended Thesis Tables

Use these tables in the thesis body or appendix:

1. `full_alm_kpi_summary.csv` as the compact headline result table.
2. `full_alm_funding_ratio_trajectory.csv` as the funding path table.
3. `full_alm_annual_cashflows.csv` as the annual liquidity table.
4. `full_alm_cashflow_by_source.csv` as an appendix or support table for
   source attribution.

inspection, but it is likely too detailed for the main thesis body.

## AAL Asset Path

The generated workflow run uses the required live AAL/ACTUS service-backed
asset path. AAL asset cashflows are still a small thesis prototype portfolio,
not a calibrated production asset allocation.

## Limitations And Interpretation Risks

- The default portfolio is a small, manually inspectable thesis example, not a
  calibrated real pension fund portfolio.
- The AAL asset cashflows are generated from a small, manually inspectable
  prototype portfolio, not from a calibrated production asset allocation.
- The Stage-1 liability is a transparent deterministic proxy and excludes
  technical reserves.
- The workflow does not include stochastic scenarios, mortality tables, wage
  growth, inflation, new entrants, dynamic rebalancing, or strategy
  optimization.
- The outputs are appropriate for illustrating the architecture and reporting
  workflow, but they should not be presented as production-grade pension fund
  modelling or empirical benchmark evidence.
